# Omuz CRM — Руководство по установке и запуску

Добро пожаловать в проект **Omuz CRM**! В этом репозитории находятся фронтенд и бэкенд части приложения. Ниже приведена подробная инструкция по настройке, первому запуску и тестированию системы.

---

## Требования к окружению

Перед началом убедитесь, что у вас установлены:
- **Node.js** (версия 18.x или выше)
- **npm** (версия 9.x или выше)
- **Git**

---

## Структура проекта

- `/backend` — Серверная часть на NestJS с базой данных SQLite (Prisma ORM).
- `/frontend` — Клиентская часть на React (Vite, TypeScript, TailwindCSS).

---

## 1. Настройка и запуск Бэкенда (backend)

1. Откройте терминал и перейдите в директорию `backend`:
   ```bash
   cd backend
   ```

2. Установите зависимости:
   ```bash
   npm install
   ```

3. Настройте файл окружения:
   Скопируйте пример файла конфигурации `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```
   *По умолчанию файл `.env` настроен на локальную базу данных SQLite (`dev.db`).*

4. Примените миграции базы данных и создайте базу данных:
   ```bash
   npx prisma db push
   ```

5. Наполните базу данных начальными (тестовыми) данными:
   ```bash
   npx prisma db seed
   ```
   *Это создаст тестовые филиалы, группы, студентов, ментора и супер-администратора.*

6. Запустите бэкенд-сервер в режиме разработки:
   ```bash
   npm run start:dev
   ```
   Сервер запустится на порту `3000` (http://localhost:3000).

---

## 2. Настройка и запуск Фронтенда (frontend)

1. Откройте новый терминал и перейдите в директорию `frontend`:
   ```bash
   cd frontend
   ```

2. Установите зависимости:
   ```bash
   npm install
   ```

3. Запустите клиентское приложение в режиме разработки:
   ```bash
   npm run dev
   ```
   Приложение запустится и будет доступно в браузере по адресу: http://localhost:5173 (или по порту, указанному в консоли).

---

## 3. Учётные данные для тестирования (Тестовые аккаунты)

После заполнения базы данных с помощью команды `npx prisma db seed`, вы можете войти в систему под следующими ролями:

### А. Супер Администратор (Super Admin)
- **Телефон (Логин):** `+992 000000000` (в поле ввода вводить: `000000000`)
- **Пароль:** `000000000`

### Б. Ментор (Mentor)
- **Телефон (Логин):** `+992 992000001` (в поле ввода вводить: `992000001`)
- **Пароль:** `password123`

### В. Тестовый Студент (Student)
- **Телефон (Логин):** `+992 200000000` (в поле ввода вводить: `200000000`)
- **Пароль:** `password123`

## Возможные проблемы и решения

- **Ошибка при запуске базы данных / Prisma:**  
  Убедитесь, что вы запустили команду `npx prisma db push` перед `npx prisma db seed`. Если база данных заблокирована или повреждена, вы можете удалить файл `backend/dev.db` и запустить процесс настройки базы данных заново.
